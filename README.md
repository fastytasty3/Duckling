# Счётчик времени обработки товаров

Веб-сервис для учёта времени обработки товаров на складе. Операторы сканируют штрихкоды — система фиксирует, сколько времени уходит на каждую единицу товара. Руководители видят всё в режиме реального времени.

---

## Для чего это нужно

На складе операторы обрабатывают товары: сортируют, маркируют, упаковывают и т.д. Без автоматизации непонятно:
- Сколько времени уходит на одну единицу?
- Кто работает эффективнее?
- Где возникают простои?

**Этот сервис решает задачу**: оператор сканирует штрихкод при начале работы с товаром и при завершении — система автоматически считает время, строит статистику и формирует отчёты.

---

## Роли

| Роль | Что может |
|------|-----------|
| **Оператор** | Сканировать товары, видеть свою статистику за смену |
| **Контролирующее лицо** | Видеть все рабочие столы в реальном времени, историю операций |
| **Администратор** | Всё то же самое + управление учётными записями, рабочими местами, принудительное закрытие операций |

## Инструкция для оператора

### 1. Начало работы
- Откройте сервис на рабочем компьютере
- Нажмите **«Авторизоваться»** и выберите своё имя из списка
- Выберите **смену** (дневная / ночная)

### 2. Сканирование товара
- Наведите сканер на штрихкод товара и отсканируйте его
- Система запустит таймер — начнётся отсчёт времени обработки
- Введите **количество единиц** в поле (если нужно)

### 3. Пауза
- Если нужно прерваться — нажмите **«Пауза»**
- Пауза учитывается отдельно и не входит в чистое время обработки
- Для продолжения нажмите **«Продолжить»**

### 4. Завершение операции
- Когда товар обработан — отсканируйте следующий штрихкод (предыдущая операция закроется автоматически) или нажмите **«Завершить»**

### 5. Статистика за смену
- На главном экране внизу видна ваша статистика: количество операций, единиц товара, среднее время на единицу

### 6. Завершение работы
- Нажмите кнопку выхода в правом верхнем углу
- Данные сохранятся автоматически

## Инструкция для контролирующего лица / администратора

### Вход
- Перейдите по адресу сервиса и нажмите **«Контроль»** (или добавьте `/supervisor` к адресу)
- Введите логин и пароль

### Вкладка «Рабочие столы»
Главный экран мониторинга — обновляется в реальном времени.

- Столы сгруппированы по **ОКиУ** (зонам)
- Каждая строка — один рабочий стол:
  - 🟢 **Активен** — оператор работает
  - ⚪ **Не активен** — никого нет
- Кликните по строке активного стола, чтобы развернуть детали: кто работает, какой товар сканируется, таймер, ФИО сотрудников
- Кнопка **«Закрыть»** — принудительно завершить текущую операцию на столе (для администраторов)

### Вкладка «История»
- Все операции с фильтрами по дате, оператору, рабочему месту, штрихкоду
- Экспорт в Excel

### Вкладка «Отчёты»
- Сводная статистика за период
- По товарам, по операторам, по часам

### Вкладка «Журнал безопасности»
- Все действия в системе: входы, выходы, принудительные закрытия операций

### Вкладка «Учётные записи» *(только администратор)*
- Создание новых контролирующих лиц
- Сброс паролей, блокировка

### Вкладка «Рабочие места» *(только администратор)*
- Добавление и редактирование рабочих столов
- Привязка к ОКиУ (зоне)

## Технические детали

- **Фронтенд**: React 19 + Vite (TypeScript)
- **Бэкенд**: Express 5 (Node.js, TypeScript)
- **База данных**: PostgreSQL (Drizzle ORM)
- **Реальное время**: WebSocket
- **Инфраструктура**: pnpm monorepo, развёрнут на Replit
