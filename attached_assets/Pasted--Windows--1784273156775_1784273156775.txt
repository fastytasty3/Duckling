Создай полноценную программу для ПК под Windows для учета времени обработки товаров с помощью сканера штрихкодов.

Главная задача

Пользователь подключает к компьютеру обычный USB-сканер штрихкодов. Сканер работает в режиме клавиатуры: вводит штрихкод и отправляет Enter.

После сканирования товара программа должна автоматически начинать отсчет времени работы с этим товаром.

Когда пользователь сканирует другой товар:

1. Таймер предыдущего товара автоматически останавливается.
2. Результат предыдущей операции сохраняется в историю.
3. Для нового товара автоматически запускается новый таймер.
4. Все изменения отображаются в реальном времени.

Пример:

* В 10:00:00 отсканирован товар 4601234567890.
* Начался таймер товара 4601234567890.
* В 10:04:25 отсканирован товар 4609876543210.
* Для первого товара сохраняется время обработки 4 минуты 25 секунд.
* Для второго товара запускается новый таймер.
* В 10:08:10 отсканирован третий товар.
* Для второго товара сохраняется время обработки 3 минуты 45 секунд.

Тип программы

Сделай настольное приложение для Windows.

Предпочтительный стек:

* Electron;

* SQLite для локальной базы данных;
* современный и понятный интерфейс;

Все данные должны храниться локально на компьютере. Интернет для работы программы не требуется.

Работа со сканером

Сканер штрихкодов определяется как обычная клавиатура.

В программе должно быть скрытое или постоянно активное поле приема штрихкода.

Требования:

* программа должна принимать штрихкод независимо от того, на какой элемент интерфейса нажал пользователь;
* окончанием сканирования считать нажатие Enter;
* после обработки штрихкода поле должно автоматически очищаться;
* ручной ввод штрихкода также должен работать;
* случайные одиночные нажатия клавиш не должны восприниматься как сканирование;
* минимальная длина штрихкода настраивается, по умолчанию 4 символа;
* поддерживать цифры и буквы;
* убрать пробелы в начале и конце кода;
* повторное быстрое получение одного и того же кода не должно случайно создавать дублирование, если с момента прошлого сканирования прошло меньше установленного времени, например 500 миллисекунд.

Логика учета времени

В каждый момент времени активным может быть только один товар.

Для активного товара отображать:

* штрихкод;
* наименование товара;
* артикул;
* время начала;
* текущее время обработки;
* количество отсканированных единиц;
* ФИО или имя оператора;
* смену;
* рабочее место.

Таймер должен обновляться каждую секунду.

При сканировании другого штрихкода:

* текущая операция завершается;
* время окончания фиксируется;
* длительность рассчитывается и сохраняется;
* новый товар становится активным;
* для нового товара запускается таймер.

Добавь кнопку «Остановить текущую операцию». Она должна завершать таймер без запуска нового товара.

Добавь кнопку «Пауза». Время паузы не должно включаться в чистое время обработки товара.

После паузы должна быть кнопка «Продолжить».

Если пользователь закрыл программу во время активного таймера, после следующего запуска программа должна показать незавершенную операцию и предложить:

* продолжить отсчет;
* завершить операцию временем закрытия программы;
* завершить операцию текущим временем;
* удалить незавершенную операцию.

Работа с одинаковыми товарами

В настройках добавь два режима.

Режим 1. Каждый скан — новая операция

Если повторно отсканирован тот же штрихкод, текущая операция завершается и сразу создается новая операция с этим же товаром.

Режим 2. Повторный скан увеличивает количество

Если повторно отсканирован тот же активный штрихкод:

* таймер не перезапускается;
* количество обработанных единиц увеличивается на 1.

Если отсканирован другой штрихкод, предыдущая операция завершается и запускается новая.

По умолчанию включить второй режим.

* единица измерения;

При первом сканировании неизвестного штрихкода программа должна:

1. Запустить таймер, чтобы не терять время.
2. Показать уведомление «Товар не найден в справочнике».
3. Предложить заполнить наименование и артикул.
4. Позволить сохранить товар в справочник.
